# Project Usage Guide: Skin Lesion Binary Classification with Selective Classification

## Overview
This project implements a clinical decision support pipeline for binary skin lesion classification (Malignant vs Benign). It features temperature scaling for model calibration, cost-aware selective classification with a reject option, and comprehensive cost-coverage analysis. The pipeline is designed to run sequentially, with each stage depending on artifacts generated by the previous stage.

## Prerequisites & Installation
- Python 3.9 or higher
- CUDA-compatible GPU (strongly recommended)
- Operating System: Linux, macOS, or Windows

Create a virtual environment and install dependencies:
```bash
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip install torch torchvision timm numpy pandas scikit-learn albumentations opencv-python matplotlib seaborn tqdm netcal
```

## Data Preparation
1. Create a `data/raw/` directory in the project root.
2. Place all ISIC 2019 dermoscopic images inside `data/raw/`.
3. Place the following metadata files in `data/raw/`:
   - `ISIC_2019_Training_Metadata.csv`
   - `ISIC_2019_Training_GroundTruth.csv`

## Pipeline Execution Steps
Execute the following commands in order. Replace `exp_001` with your preferred experiment identifier if desired.

### Step 1: Data Splitting
Splits the dataset into train, validation, and test sets while preventing patient leakage.
```bash
python src/data/splitter.py --exp_id exp_001
```
**Outputs:** `experiments/exp_001/data/exp_001_catalog.csv`, `experiments/exp_001/data/exp_001_manifest.json`

### Step 2: Model Training
Trains a convolutional or transformer-based classifier on the training split.
```bash
python src/train.py \
  --catalog experiments/exp_001/data/exp_001_catalog.csv \
  --manifest experiments/exp_001/data/exp_001_manifest.json \
  --img_dir data/raw \
  --output_dir experiments \
  --model_type vit \
  --batch_size 64 \
  --epochs 50 \
  --learning_rate 1e-4 \
  --dropout 0.5
```
Supported `--model_type` values: `vit`, `resnet50`, `resnet101`, `resnet152`, `densenet121`, `densenet169`, `convnext_tiny`, `convnext_base`.
**Outputs:** `experiments/exp_001/models/best_checkpoint.pth`, `experiments/exp_001/models/train_config.json`

### Step 3: Inference Generation
Generates and caches model predictions for all splits.
```bash
python src/run_inference.py \
  --catalog experiments/exp_001/data/exp_001_catalog.csv \
  --manifest experiments/exp_001/data/exp_001_manifest.json \
  --checkpoint experiments/exp_001/models/best_checkpoint.pth \
  --config experiments/exp_001/models/train_config.json \
  --all_splits \
  --cache_dir cache
```
**Outputs:** Cached `.npz` and `.json` files in `cache/`

### Step 4: Temperature Calibration
Optimizes a temperature parameter on the validation set to align predicted probabilities with true likelihoods. Finds a threshold targeting a specific specificity.
```bash
python src/calibrate.py \
  --catalog experiments/exp_001/data/exp_001_catalog.csv \
  --manifest experiments/exp_001/data/exp_001_manifest.json \
  --checkpoint experiments/exp_001/models/best_checkpoint.pth \
  --config experiments/exp_001/models/train_config.json \
  --cache_dir cache \
  --target_specificity 0.95
```
**Outputs:** `experiments/exp_001/calibration/calibration.json`

### Step 5: Test Set Evaluation
Evaluates calibrated model performance on the held-out test set.
```bash
python src/evaluate.py \
  --catalog experiments/exp_001/data/exp_001_catalog.csv \
  --manifest experiments/exp_001/data/exp_001_manifest.json \
  --checkpoint experiments/exp_001/models/best_checkpoint.pth \
  --calibration experiments/exp_001/calibration/calibration.json \
  --cache_dir cache
```
**Outputs:** `experiments/exp_001/evaluation/test_evaluation.json`

### Step 6: Selective Classification Experiments
Runs cost-aware selective classification across thousands of cost configurations. Uses parallel processing.
```bash
python src/selective_classification.py \
  --checkpoint experiments/exp_001/models/best_checkpoint.pth \
  --manifest experiments/exp_001/data/exp_001_manifest.json \
  --calibration experiments/exp_001/calibration/calibration.json \
  --split test \
  --cache_dir cache
```
Optional: Provide a custom cost JSON with `--config_file path/to/configs.json`.
**Outputs:** `experiments/exp_001/selective_classification/selective_results_test_expanded.csv`, `selective_summary_test_expanded.json`

## Cost Configuration Management
The system supports synthetic and clinically grounded cost structures.

**Generate Synthetic Cost Configurations (Normalized by False Negative cost):**
```bash
python src/configs/selective_cost_configs.py --output configs/selective_configs_normalized.json
```

**Generate Real-World Clinical Cost Configurations (Fixed referral cost):**
```bash
python src/configs/real_life_configs.py --output configs/real_life_dermatology_configs.json
```

Pass these files to `src/selective_classification.py` using the `--config_file` argument.

## Important Notes
1. **Sequential Execution:** The pipeline must be run in the order presented. Each script reads artifacts created by the previous step.
2. **Caching:** Inference results are cached in `cache/`. To regenerate predictions after model or data changes, add `--force` to `src/run_inference.py`.
3. **Parallel Processing:** Analysis scripts use `ProcessPoolExecutor`. Adjust worker counts or system limits if encountering memory constraints on large datasets.
4. **Cost Normalization:** All cost configurations are normalized by the False Negative cost by default. This ensures comparability across different clinical risk tiers.
5. **Clinical Thresholds:** The selective classification logic uses closed-form threshold equations. If referral costs exceed error costs, the system defaults to standard binary classification (threshold = 0.5).
